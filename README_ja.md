# Local RTMP Server (macOS) v3.0

軽量、ネイティブ、高性能なmacOS専用RTMPサーバーです。SwiftUIとNode Media Serverを利用して構築されており、ローカルネットワーク内でストリーミングサーバーを簡単に立ち上げることができます。画面配信、ストリーミングソフトウェア（OBSなど）のテスト、ローカルネットワークでの映像フィード転送に最適です。

## ✨ v3.0 の新機能

- **macOS ネイティブインターフェース**: SwiftとSwiftUIで完全に再構築されました。すりガラス効果 (macOS 13+) を備えた、美しくモダンで非常にスムーズなネイティブUIをお楽しみください。
- **動的マルチIPサポート**: マシンの全てのアクティブなIPv4ネットワークインターフェースを自動的に検出して表示します。異なるネットワーク (Wi-Fi、Ethernet) の正確なRTMP URLを即座に確認してコピーできます。
- **ライブ HLS プレビュー**: ゼロ遅延のネイティブAVPlayerプレビューウィンドウを内蔵。OBSからストリーミングを開始すると、アプリ内で即座に映像を監視できます。
- **スマート設定エンジン**: 配信キーやポートをいつでも安全に変更できます。ストリーミング中に設定を保存した場合、サーバーはストリーミングが終了するまでネットワークの再起動をインテリジェントに遅延させ、予期せぬ切断を防ぎます。
- **メニューバーとDockモード**: サーバーを「メニューバー」でバックグラウンドで静かに実行するか、Dock上の標準アプリケーションとして実行するかを選択できます。
- **自動起動**: システム起動時にアプリを自動的に起動し、すぐにRTMPサーバーを起動するように設定できます。
- **多言語サポート**: 英語、繁体字中国語 (繁體中文)、日本語、スペイン語 (Español)、フランス語 (Français) を完全にサポートしています。

## 🚀 インストール

1. Releases ページから最新の `Local RTMP Server 3.0.dmg` をダウンロードします。
2. DMGファイルをダブルクリックしてマウントします。
3. **Local RTMP Server** アプリのアイコンを **アプリケーション (Applications)** フォルダにドラッグします。
4. Launchpad またはアプリケーションフォルダからアプリを起動します。

> **注意**: macOS で「開発元が未確認のため開けません」というセキュリティ警告が表示された場合は、**システム設定 > プライバシーとセキュリティ** に移動し、**このまま開く** をクリックしてください。

## 📖 使い方

1. **サーバーを起動**: アプリの「再生 (Play)」ボタンをクリックすると、ステータスランプが緑色に変わります。
2. **RTMP URL をコピー**: アプリに現在のローカルIPアドレスが表示されます。そのURLをコピーしてください (例: `rtmp://192.168.1.100/live/mystreamkey`)。
3. **OBS の設定**:
   - OBSの 設定 -> 配信 に移動します。
   - サービス: `カスタム (Custom)`
   - サーバー: `rtmp://192.168.1.100/live`
   - ストリームキー: `mystreamkey`
4. **配信開始**: OBSで「配信開始」をクリックします。
5. **プレビュー**: アプリの「ライブプレビュー」ボタンをクリックすると、ストリーミング画面をリアルタイムで監視できます。

## 🛠 詳細設定
`Cmd + ,` を押すか、歯車アイコンをクリックして「設定」を開きます。
- **ストリームキータイプ**: 覚えやすい固定キーを使用するか、アプリが毎回安全なランダムキーを自動生成するかを選択します。
- **カスタムポート**: 他のサービスと競合する場合は、デフォルトのRTMPポート (1935) または HTTP HLS ポート (8000) を変更できます。
- **アプリの表示モード**: アプリをバックグラウンド (メニューバー) で完全に実行するか、Dock 上の標準アプリとして表示するかを切り替えます。
